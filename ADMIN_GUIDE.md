# Админ-панель MTS Cloud

## Доступ к админ-панели

Админ-панель доступна по адресу: **http://localhost:3003/admin**

### Учетные данные для входа:
- **Логин:** admin
- **Пароль:** 12345

## Возможности админ-панели

### 1. Дашборд со статистикой
- Общее количество пользовательских сессий
- Количество активных сессий
- Общее количество виртуальных машин
- Средняя длительность сессии

### 2. Графики и аналитика
- **График активности по часам** - показывает распределение активности пользователей в течение суток
- **График по платформам** - круговая диаграмма использования различных облачных платформ (AWS, Azure, Google Cloud)

### 3. Управление сессиями пользователей
- Просмотр всех активных и неактивных сессий
- Поиск по имени пользователя, email или IP-адресу
- Фильтрация по статусу (все/активные/неактивные)
- Информация о каждой сессии:
  - Имя пользователя и email
  - IP-адрес
  - Используемая платформа
  - Количество VM
  - Время входа
  - Длительность сессии
  - Статус (активна/неактивна)

### 4. Действия с сессиями
- **Завершить сессию** - деактивирует активную сессию пользователя
- **Удалить сессию** - полностью удаляет запись о сессии
- **Массовые операции** - возможность завершить или удалить несколько сессий одновременно

## Структура файлов

```
src/
├── Admin.tsx                    # Главный компонент админ-панели
├── contexts/
│   └── SessionContext.tsx       # Контекст для управления сессиями
└── components/
    ├── AdminLogin.tsx          # Компонент входа в админ-панель
    └── AdminPanel.tsx          # Основная панель управления
```

## Технологии

- **React Router** - для роутинга (/admin)
- **Framer Motion** - для анимаций
- **Recharts** - для графиков и диаграмм
- **Context API** - для управления состоянием сессий
- **localStorage** - для сохранения данных о сессиях
- **sessionStorage** - для хранения статуса аутентификации админа

## Особенности

1. **Автоматическая генерация тестовых данных** - при первом запуске создаются фейковые сессии пользователей для демонстрации
2. **Автообновление** - статус активности сессий обновляется каждые 30 секунд
3. **Сохранение данных** - все изменения сессий сохраняются в localStorage
4. **Адаптивный дизайн** - панель адаптирована для различных размеров экранов
5. **Защита паролем** - доступ только с правильными учетными данными

## Безопасность

⚠️ **Важно:** Это демонстрационная версия. В production-окружении необходимо:
- Использовать безопасное хранение паролей (хеширование)
- Реализовать JWT-токены для аутентификации
- Добавить защиту от CSRF-атак
- Использовать HTTPS
- Добавить rate limiting для предотвращения брутфорса
- Реализовать серверную валидацию и авторизацию
