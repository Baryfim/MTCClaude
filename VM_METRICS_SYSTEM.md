# Система мониторинга метрик VM

## Обзор
Реализована система автоматического получения метрик для всех активных виртуальных машин каждые 10 секунд.

## Компоненты

### 1. Типы данных (`src/types.ts`)
- `VMMetrics` - интерфейс для метрик VM (cpu_cores, ram_mb, status, storage)
- `VMMetricsHistory` - история метрик с timestamp
- `VMMetricsTimeSeries` - данные для графиков

### 2. Redux Slice (`src/lib/slices/userVMsSlice.ts`)
**Новые async thunks:**
- `fetchVMMetricsAsync` - получение метрик для одной VM
- `fetchAllVMMetricsAsync` - получение метрик для всех активных VM

**Новое состояние:**
- `vmMetrics` - текущие метрики каждой VM
- `metricsHistory` - история метрик (последние 100 записей на VM)

**Новые actions:**
- `updateVMMetrics` - обновление метрик VM
- `clearMetricsHistory` - очистка истории

**Новые селекторы:**
- `selectVMMetrics` - все метрики
- `selectMetricsHistory` - история метрик
- `selectVMMetricsById` - метрики конкретной VM
- `selectVMMetricsHistoryById` - история метрик конкретной VM

### 3. Хук для опроса (`src/lib/hooks/useVMMetricsPolling.ts`)
**Функциональность:**
- Автоматически запрашивает метрики каждые 10 секунд
- Работает только для активных VM (статус RUNNING)
- Запускается/останавливается в зависимости от параметра `enabled`
- Автоматически очищает интервалы при размонтировании

**Использование:**
```typescript
useVMMetricsPolling(enabled, interval);
// enabled - включить/выключить опрос
// interval - интервал в миллисекундах (по умолчанию 10000)
```

### 4. Компонент Charts (`src/blocks/Charts/Charts.tsx`)
**Обновления:**
- Теперь использует реальные данные из Redux store
- Автоматически отображает метрики из истории
- Поддерживает фильтрацию по конкретной VM через проп `vmId`
- При отсутствии данных показывает mock-данные

**Новый проп:**
- `vmId?: string` - опционально для отображения метрик конкретной VM

### 5. Интеграция в App (`src/App.tsx`)
- Хук `useVMMetricsPolling` активируется когда пользователь залогинен и находится на дашборде
- Компонент `Charts` обновлен для работы без пропса `data`

## API эндпоинты

### Получение метрик VM
```
GET /v1/resources/{id}/metrics/
```

**Ответ:**
```json
{
  "cpu_cores": 0,
  "ram_mb": 0,
  "status": "CREATING" | "RUNNING" | "STOPPED" | "ERROR" | "DELETING",
  "storage": 0
}
```

## Особенности реализации

1. **Автоматический опрос**: Метрики запрашиваются автоматически каждые 10 секунд для всех активных VM
2. **Умная история**: Хранятся последние 100 записей метрик для каждой VM
3. **Обработка ошибок**: Ошибки при получении метрик логируются, но не блокируют работу приложения
4. **Оптимизация**: Параллельные запросы для всех VM (Promise.all)
5. **Очистка данных**: Метрики автоматически удаляются при удалении VM
6. **Mock-данные**: При отсутствии реальных данных показываются mock-данные для UX

## Примеры использования

### В компонентах
```typescript
import { useAppSelector } from '@/lib/hooks';
import { selectVMMetrics, selectVMMetricsById } from '@/lib/slices/userVMsSlice';

// Получить все метрики
const allMetrics = useAppSelector(selectVMMetrics);

// Получить метрики конкретной VM
const vmMetrics = useAppSelector(selectVMMetricsById('vm-123'));
```

### Отображение графиков
```typescript
// Для всех VM
<Charts />

// Для конкретной VM
<Charts vmId="vm-123" />
```

## Конфигурация

Интервал опроса можно изменить при вызове хука:
```typescript
// Опрос каждые 5 секунд
useVMMetricsPolling(enabled, 5000);

// Опрос каждую минуту
useVMMetricsPolling(enabled, 60000);
```
